# Issue tracker: local Markdown

Issues and specs live in the repository-level `.scratch/` (`../.scratch/` from `frontend-v2/`):

- One feature per `.scratch/<feature-slug>/` directory.
- The spec is `.scratch/<feature-slug>/spec.md`.
- Implementation tickets live under `.scratch/<feature-slug>/build-tickets/`.
- Architecture investigations live under `.scratch/<feature-slug>/issues/`.
- A `Status:` line records the current state; comments append under `## Comments`.

For the frontend rewrite, `spec.md`, `map.md`, `gaps-and-recommendations.md`, `additional-scan-findings.md`, and build ticket 01's standing rules are binding for every ticket.
